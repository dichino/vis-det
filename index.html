About Vis Det

Vis Det – Impact Journal

A local-first prototype for structured impact documentation. It helps teams document projects, observations, evidence, people reached and lightweight survey feedback in one place.

Why this exists

The prototype is inspired by Norge Unlimited's work with local social entrepreneurship, neighbourhood incubators and impact documentation. It explores how early impact data can be captured before it becomes formal reporting.

Journal evidence

Document evidence

Log entries with people reached, tags, and optional files so progress can be reviewed later.

People reached

Track reach

Surveys and QR

Collect responses

Local-first data

Prototype disclaimer

This is a local-first prototype for exploring impact documentation. It is not an official production system for Norge Unlimited.

Project Notes

Built as a focused app prototype.

A small, inspectable MVP for recruiters and technical reviewers: plain files, local storage, real import/export flows and no fake service layer.

Built by

Dichino Nguyen as an app development prototype for Norge Unlimited.

Tech stack

  • HTML, CSS and vanilla JavaScript
  • Dexie.js with IndexedDB
  • Local JSON import and export

Prototype scope

  • Project-based journal entries
  • Tags, attachments and people reached
  • Survey builder, QR sharing flow and response imports

Current limits

  • Local-only data
  • No authentication, backend or multi-user sync
  • Not a production CRM or reporting system

Architecture

Projects, entries, surveys and responses are stored locally in IndexedDB with Dexie.js. Export/import uses JSON. A production version would need backend sync, authentication and role-based access before supporting shared teams.

Possible next steps

Backend sync, user accounts, role-based access, CSV/PDF exports, hosted survey pages, analytics views and admin reporting could be explored after validating the core documentation workflow.
