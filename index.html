About Vis Det

Vis Det – Impact Journal

Vis Det turns scattered project notes, feedback and evidence into structured reporting material.

Why this exists

The prototype is inspired by Norge Unlimited's work with local social entrepreneurship, neighbourhood incubators and impact documentation. It explores how early impact data can be captured before it becomes formal reporting.

Journal evidence

Document evidence

Log entries with people reached, tags, and optional files so progress can be reviewed later.

People reached

Track reach

Keep simple counts close to the qualitative notes that explain what changed.

Surveys and QR

Collect responses

Create lightweight surveys, export them as standalone HTML, and import response JSON files.

Local-first data

Stay local

Data is stored in this browser with IndexedDB. Export and import are explicit file actions.

How it works

1 Document activities

Write short notes from workshops, meetings and follow-ups.

2 Add evidence and reach

Record people reached, tags and optional attachments.

3 Collect feedback

Use lightweight surveys and import responses locally.

4 Summarize impact

Export data or draft a report-ready summary from local evidence.

Prototype disclaimer

This is a local-first prototype for exploring impact documentation. It is not an official production system for Norge Unlimited.

Project Notes

Built as a focused app prototype.

A focused local-first prototype for social entrepreneurs and support teams documenting impact: validation, IndexedDB modeling, reporting logic and export workflows.

Built by

Dichino Nguyen as an app development prototype for Norge Unlimited.

Tech stack

  • HTML, CSS and vanilla JavaScript
  • Dexie.js with IndexedDB
  • Dexie-backed local-first data model
  • Static GitHub Pages deployment target

Prototype scope

  • Project-based journal entries
  • Tags, attachments and people reached
  • Survey builder, QR sharing flow and response imports
  • Import Wizard with validation and result summary
  • Evidence Readiness scoring and Timeline view
  • Impact Summary, AI-ready prompt and report package export
  • English/Norwegian language toggle

Current limits

  • Local-only data
  • No authentication, backend or multi-user sync
  • No external AI service or exposed API keys
  • Not a production CRM or reporting system

Architecture

Projects, entries, surveys and responses are stored locally in IndexedDB with Dexie.js. Export/import uses JSON. A production version would need backend sync, authentication and role-based access before supporting shared teams.

Possible next steps

Backend sync, user accounts, role-based access, CSV/PDF exports, hosted survey pages, analytics views and admin reporting could be explored after validating the core documentation workflow.
